## 1. 目的

这个目录用于保存项目开发过程中的系统日志。

它不是代码注释，也不是普通需求文档。  
它的目标是让未来回顾项目时，能快速知道：

- 当时聊过什么
- 做过什么决策
- 为什么按这个顺序开发
- 哪些验证已经通过
- 下一步应该接什么

---

## 2. 重要限制

Codex 当前不能保证自动导出完整逐字聊天记录。

因此这里保存的是结构化开发日志：

- 关键对话摘要
- 重要决策
- 开发节点
- 验证结果
- commit 记录
- 下一步计划

如果未来有原始聊天导出能力，可以再把逐字 transcript 放进这个目录。

---

## 3. 日志命名规则

每个日志文件建议使用：

```text
YYYY-MM-DD_主题.md
```

例子：

```text
2026-05-26_M1基础闭环与UI校准.md
```

---

## 4. 写入规则

每次重要开发节点结束后，至少记录：

- 当前目标
- 用户反馈
- 已完成工作
- 验证命令
- commit
- 下一步

日志必须自包含。  
不要写“见某某文档”作为唯一信息来源。
