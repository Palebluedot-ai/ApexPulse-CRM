# M1.22A 手机端 Dogfood Runbook

## 1. 目的

这份文档帮助 Chao 用手机跑第一轮真实 dogfood。

这份文档是自包含文档。

阅读这份文档不需要依赖其他文档。

第一轮手机端 dogfood 的目标是：

```text
手机上传 1 张真实微信截图 -> AI 提取 -> Review 人工确认 -> 客户入库 -> 任务生成
```

第一轮不要一次上传很多截图。

推荐节奏：

```text
第 1 轮：1 张真实截图
第 2 轮：3 到 5 张真实截图
第 3 轮：10 到 20 张真实截图
```

## 2. 当前边界

当前允许：

- 使用真实微信截图
- 把截图发送到已配置的外部视觉 API
- 在本地数据库里创建真实客户和任务
- 人工修正 AI 提取结果

当前不做：

- 不自动入库
- 不批量确认
- 不批量删除
- 不自动合并客户
- 不做团队权限
- 不做云部署

## 3. Mac 启动步骤

在 Mac 上打开终端，进入项目目录：

```bash
cd /Users/chao/Projects/hashkey-otc-crm-v1
```

启动 Postgres：

```bash
docker compose up -d postgres
```

启动开发服务：

```bash
pnpm dev
```

终端会显示类似：

```text
Local:   http://localhost:3000
Network: http://10.x.x.x:3000
```

手机要打开的是 `Network` 那一行，不是 `localhost`。

## 4. 手机访问条件

手机端访问需要满足：

- Mac 和手机连同一个 Wi-Fi
- Mac 上 `pnpm dev` 正在运行
- 手机浏览器打开的是 `Network` 地址
- Mac 没有被防火墙拦住本地网络访问

手机浏览器打开：

```text
http://10.x.x.x:3000
```

这里的 `10.x.x.x` 要替换成终端里显示的 Network 地址。

## 6. 第一轮手机真实测试路径

### 6.1 打开录入页

手机浏览器访问：

```text
http://10.x.x.x:3000/capture
```

### 6.2 上传 1 张微信截图

在“上传截图 / 名片照片”区域：

1. 点击选择图片。
2. 从手机相册选择 1 张真实微信截图。
3. 可选填写备注。
4. 点击保存图片证据。

成功后应该看到：

- 待确认事件 ID
- 本轮已送入待确认数量
- 继续上传下一张
- 去待确认处理

第一轮建议直接点击：

```text
去待确认处理
```

### 6.3 Review

进入：

```text
http://10.x.x.x:3000/review
```

找到刚上传的截图。

操作顺序：

1. 检查原图预览。
2. 点击 AI 提取字段。
3. 等待字段回填。
4. 检查客户名、公司、来源、需求、下一步、下次跟进时间。
5. 修正 AI 识别错的字段。
6. 如果是新客户，保留“新客户 / 暂不匹配现有客户”。
7. 如果是老客户，选择对应客户。
8. 点击确认入库。

AI 提取只是回填字段。

AI 不会自动入库。

### 6.4 检查客户详情

确认入库后进入：

```text
http://10.x.x.x:3000/customers
```

检查：

- 是否出现新客户
- 客户名是否正确
- 最新沟通摘要是否正确
- 跟进状态是否合理
- 客户详情第一屏是否能看懂下一步

### 6.5 检查任务

进入：

```text
http://10.x.x.x:3000/tasks
```

检查：

- 是否出现自动生成的 follow-up task
- task 描述是否来自下一步
- due date 是否合理
- 任务可以完成或恢复

## 7. 第二轮连续测试路径

第一轮 1 张截图跑通后，再做第二轮。

第二轮建议：

```text
连续上传 3 到 5 张真实微信截图。
```

操作方式：

1. 每次上传 1 张。
2. 上传成功后点“继续上传下一张”。
3. 上传完 3 到 5 张后，再去 Review。
4. 在 Review 里逐条 AI 提取和确认。
5. 不要批量确认。

这样做的原因：

- 每张图都有独立 event id
- 错误更容易定位
- 不会因为一张图识别错影响其他图

## 8. 常见问题

### 8.1 手机打不开页面

先检查：

- 手机和 Mac 是否同一个 Wi-Fi
- 是否打开的是 Network URL
- Mac 上 `pnpm dev` 是否还在运行
- Mac 防火墙是否拦截

不要在手机上打开：

```text
http://localhost:3000
```

手机上的 localhost 是手机自己，不是 Mac。

### 8.2 上传失败

先检查：

- 是否选择了图片
- 图片格式是否是截图或照片
- 网络是否断开
- Mac 终端是否有报错

如果失败，不要重复上传很多次。

先记录失败截图和页面提示。

### 8.3 AI 提取失败

AI 提取失败时：

- 原始截图仍然保留
- event 仍然在待确认队列
- 可以手工填写字段
- 可以稍后重新点 AI 提取

不要因为 AI 失败就删除数据。

### 8.4 AI 识别错客户名

这是预期内风险。

处理方式：

- 人工改客户名
- 检查公司和来源
- 确认后再入库

如果已经确认错了，先记录 customer id 和 event id。

不要直接删数据库。

## 9. 第一轮通过标准

第一轮通过标准：

- 手机能打开 CRM
- 手机能登录
- 手机能上传 1 张真实微信截图
- 图片进入 Review
- AI 能提取字段
- 用户能人工修正
- 用户能确认入库
- 客户详情能看到最新沟通
- 任务页能看到 follow-up task

如果这些都满足，就可以进入第二轮 3 到 5 张连续测试。
