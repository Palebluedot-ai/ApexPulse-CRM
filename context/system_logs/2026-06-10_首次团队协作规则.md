## 4. 规则重点

首次团队协作规则覆盖：

- 开始前先 `git status -sb` 和 `git pull --ff-only`
- 不使用 destructive git 命令
- 不提交 secret、真实截图、`data/`
- 不擅自改变产品边界
- 不擅自删除数据库数据或重做 migration
- 两个人避免同时改同一批文件
- 业务规则先写测试
- 完成后运行 `pnpm check`
- 涉及部署或 Next route 时运行 `pnpm build`
- 文档必须自包含
- 另一个大模型进入项目时必须先读项目规则和协作规则
