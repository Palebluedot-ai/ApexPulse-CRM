## 4. 已完成开发

已新增本地图片存储 helper。

helper 负责：

- 清洗文件名
- 校验图片 MIME type
- 校验正数文件大小
- 生成本地 storage key
- 防止本地路径穿越
- 写入本地附件目录

图片保存目录：

```text
data/attachments/
```

该目录已被 `.gitignore` 忽略。

已改造图片 Capture API。

`/api/capture/image` 已从 JSON metadata 改成 multipart form upload。

前端提交：

- `imageFile`
- `note`

后端会：

- 接收真实图片文件
- 保存到本地
- 创建 pending review event
- 创建 attachment 记录

已新增附件读取 API。

```text
GET /api/attachments/:id
```

这个接口只读取安全的本地附件。

如果 attachment 不是本地 storage key，会返回 404。

已更新 Capture 页面文案。

页面现在明确说明：

- 当前是真实本地上传
- 图片会保存到 `data/attachments/`
- 先不接 OCR
- 后续在 Review 人工确认字段

已更新 Review 页面附件区。

本地图片附件会显示：

```text
打开图片证据
```

旧的占位附件不会显示这个入口。
