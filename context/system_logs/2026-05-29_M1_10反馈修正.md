# 2026-05-29 M1.10 反馈修正系统日志

## 1. 本轮背景

Chao 早上打开当前页面后提出产品反馈。

核心反馈：

- 首页底部三个输入卡片没有实际意义，显得乱
- Capture 页图片证据元数据不应该让用户手动输入
- Review 页不应该暴露 JSON 字段
- “不绑定客户”文案不准确
- Tasks 页任务列表应该放左边，创建任务表单应缩小
- 需要一个更全面的后续方案，包括未来 login、真实上传、Review 形态

## 2. 已执行修正

首页：

- 删除三个无实际输入能力的卡片

Capture：

- “图片证据元数据”改成“上传截图 / 名片照片”
- 用户通过文件选择器选择图片
- 文件本体暂时不上传
- 系统自动读取文件名、类型和大小
- 仍然保存待确认记录

Review：

- 删除用户可见 JSON 输入框
- 只让用户调整摘要、选择客户、选择跟进状态
- “不绑定客户”改成“新客户 / 暂不匹配现有客户”
- 技术字段继续由系统内部保留

Tasks：

- 任务列表改到左侧
- 创建任务表单改到右侧
- 创建任务表单高度降低

文档：

- 新增 `context/15_M1_10反馈修正与后续方案.md`

## 3. 当前解释

Capture 图片区域之前确实是开发占位。

它的作用是先验证附件 metadata 和事件流转。

它不是最终产品形态。

最终产品应是用户直接上传截图或名片照片，系统自动保存文件、提取内容并进入 Review。

Review 页里的 JSON 也是开发阶段残留。

业务用户不应该看到 JSON。

## 4. 当前仍未做

本轮没有做：

- login
- 真实文件上传
- OCR
- Vision API
- 真实 API key
- 真实客户数据
- migration
- 云部署
- 团队权限

## 5. 下一步建议

下一步建议进入 M1.11：Review 自然字段化。

目标是把 Review 从“摘要确认”升级为“自然字段确认”。

字段包括：

- 客户名
- 公司
- 来源
- 需求
- 下一步
- 下次跟进时间

仍然不展示 JSON。
