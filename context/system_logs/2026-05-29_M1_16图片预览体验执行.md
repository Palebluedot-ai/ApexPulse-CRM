# 2026-05-29 M1.16 图片预览体验执行

## 本轮目标

本轮目标是完成个人版真实 dogfood 前的图片证据预览体验。

这一步不接外部 AI API，不调用 OCR，也不把图片自动识别入库。

本轮只解决一个问题：

用户上传截图或名片照片之后，必须能在待确认页面直接看到图片证据，确认这条记录来自哪张图。

## 已确认的产品事实

API provider 已经由 Chao 找好。

API key 已经具备。

未来真实截图允许发送给外部 API 做识别。

这个事实只影响后续 AI 提取阶段，不改变当前 M1.16 的边界。

当前系统仍然坚持 review-first：

- 图片先保存为原始证据
- AI 识别结果以后也必须进入待确认
- 用户确认后才写入客户跟进状态
- 不做高置信自动入库

## 本轮不做的事

本轮不做：

- 不读取真实 API key
- 不新增 `.env` 中的真实密钥
- 不调用外部 API
- 不上传真实客户截图
- 不改变数据库表结构
- 不删除任何本地数据
- 不把 Review 改成自动确认

## 已完成内容

上传页 `/capture` 已增强：

- 用户选择本地图片后，会先看到本地预览
- 图片未提交前不会进入数据库
- 上传成功后，会提示去待确认页面处理
- 上传成功后，会提供“去待确认处理这张图片”的入口
- 页面文案明确说明截图和名片会先保存到本机，识别结果仍然需要 Review 确认

待确认页 `/review` 已增强：

- 本地图片附件会显示内联缩略图
- 本地图片附件可以点击打开原图
- 非本地图片或历史占位附件不会显示坏图
- 非本地附件会显示清楚的人话提示：这份附件不是本地图片，暂时只能保留文件记录

后端 Review page model 已增强：

- 每个附件会标记是否可以内联预览
- 每个附件会给出不可预览原因
- 只有本地保存的图片附件才会生成可访问预览地址
- 非本地附件不会伪装成可预览图片
