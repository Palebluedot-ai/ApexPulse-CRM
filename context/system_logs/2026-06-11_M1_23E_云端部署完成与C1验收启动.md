```bash

```

结果：

- Database: postgres

- Prepared statements: false
- Required tables: users, parties, events, attachments, tasks
- Missing tables: none

关键注意事项（踩坑记录）：

- `pnpm db:migrate` 裸跑不会自动读 `.env.local`，drizzle.config.ts 通过 `getMigrationDatabaseUrl()` 读 `process.env`，fallback 是 localhost。
- 对 Supabase 执行 migration 必须显式传入连接串。
- Migration 连接必须用 session 模式（端口 5432），不能用 transaction pooler（端口 6543），否则 prepared statements 相关操作会失败。
- 运行时连接（业务查询）继续使用 transaction pooler（端口 6543）+ `DATABASE_PREPARE=false`，这是正确的。

## 过程中发现的问题与解法

### 登录 500 问题

症状：假凭据返回 401（正常），真实凭据返回 500。

根因：Vercel 未配置环境变量。`getLocalAuthConfig()` 使用默认值（email: `chao.local@example.com`, password: `local-dev-password`），恰好与本地 `.env.local` 默认值匹配，导致密码验证通过后进入数据库查询步骤，而 `DATABASE_URL` 未设置导致 fallback 到 localhost，连接失败抛 500。

解法：在 Vercel Dashboard 补齐全部 15 个环境变量并 Redeploy。

### Migration 连接模式踩坑

症状：migration 需要 prepared statements 支持，但 Supabase transaction pooler（端口 6543）不支持。

解法：migration 使用 session 模式（同主机，端口改为 5432）。运行时连接保持 6543 + `DATABASE_PREPARE=false` 不变。
