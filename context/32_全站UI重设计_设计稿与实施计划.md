## 2. 背景:为什么重设计

第一轮真实 dogfood(Chao 本人,Mac + 手机模拟器)结论:

- 待确认页体验最差:一张卡 ~25 个元素,手机滑 4-5 屏,"审查"被做成了"编辑"。
- M1.24A 已上线捕获即自动 AI 提取(文字 + 图片),但界面还是手动时代的形态。
- 用户(Chao)明确方向:**所有内容 AI 自动提取、自动预归类,人只做轻审查。**

## 3. 设计语言

延续现有纸感,但收紧为一套 token:

```text
纸面     #faf4e8 (paper) / #f3ead8 (paper-deep) / #fffdf7 (card)
墨字     #1f1c17 (ink) / #6b6357 (ink-soft) / #e4d9c3 (line)
墨绿     #2f5d50 (tea) / #234739 (tea-deep)   → 主操作按钮、确认、当前态
柿色     #d4622a (persimmon)                   → 热度 HOT、待确认计数、警示数字
金       #b98a2f (gold)                        → WARM、提取中
红       #c2452d (red)                         → 逾期
字体     标题 Noto Serif SC / 数字 Fraunces / 正文 Noto Sans SC
```

状态色点(客户跟进状态,全站统一):红=逾期 黄=临近 绿=健康 灰=沉睡。

## 4. 七页设计决策

| 页 | 现状 | 改成 |
|---|---|---|
| ① 首页 | 5 个入口链接门户 | 今日行动清单:逾期置顶 + 今日任务直接打勾 + 待确认计数卡 + 本周三数字 |
| ② 录入 | 两个孤立表单 | 桌面:拖拽/粘贴连传;每条实时显示提取状态"✓已提取·Florian"。手机:两个大按钮(选截图/写备注) |
| ③ 待确认 | 编辑表单海 | 决策卡:缩略图+客户名+热度+AI摘要+3关键字段+绑定预判,操作[✓确认][✎修改][跳过];修改才展开完整表单;键盘 Y/E/S;提取中=骨架屏。手机:聚焦模式,一屏一卡,确认自动滑入下一张,顶部进度条 |
| ④ 客户列表 | 大卡片宫格 | 信息行(一屏12+),状态色点,默认按"该跟进程度"排序:逾期>今天>计划中>沉睡;segmented 筛选 |
| ⑤ 客户详情 | 信息平铺 | 警示条置顶("已逾期2天"+主按钮在条内);时间线带证据缩略图;待办 checklist;Morning Brief 保留 |
| ⑥ 任务 | 4 字段表单 | 一句话建任务("周五给刘总发材料"AI 解析客户+时间,复用提取管线);按紧迫分组:逾期/今天>本周>以后>已完成折底 |
| ⑦ 周报 | 列表回顾 | 大数字行(触达/完成/新增/遗留);触达列表带热度变化箭头;⚠失速信号(本周0沟通且有欠账);遗留待办一键移入下周 |

## 5. 手机端全局规范

- 底部 Tab 栏:今日 / 客户 / **＋(中央凸起大按钮=录入)** / 待确认(角标) / 任务
- **没有拍照功能**。录入 = 从相册选聊天截图(可多选)或写文字备注。用户不拍照。
- 待确认角标显示未审查数,柿色。
- 详情页主操作按钮 sticky 在底部。

## 6. 已拍板的决策(不要反复)

- AI 提取全自动(捕获即提取,M1.24A 已上线),手动"AI 提取字段"按钮只在提取缺失/失败时出现(重试用)。
- 红线不变:AI 只预填,人工点确认才入库。
- 决策卡折叠态只留 6 个元素(缩略图/客户名/摘要/≤3关键字段/绑定提示/操作钮),其余进"修改"展开层。**热度 chip 已砍**(2026-06-10 R1 dogfood 反馈);绑定预判是纯文字提示不可点,改错走 ✎修改 里的客户搜索框(datalist,留空=新建)。编辑区无热度/置信/跟进状态下拉,每个输入框带小字 label。
- 桌面键盘流:Y 确认 / E 修改 / S 跳过。**键盘提示只在页头出现一次,不重复挂在每张卡上**;操作按钮保持卡底位置。
- 中央 FAB 是"＋ 录入",不是相机图标。
- **手机周报入口**:周报不占底部 Tab 位,从今日页"本周"卡右上"看周报 →"进入,周报页头带 ← 返回。
- **任务点勾即完成**:不弹确认,勾选瞬间划线沉入"已完成"组;点错勾回来即恢复。
- **已完成任务保留 3 天**:列表里可见 3 天(回查/反悔窗口),之后自动从列表清掉;数据库不删,客户时间线仍可追溯。
- **客户详情已完成待办**:勾掉后划线收进"▸ 已完成 N 条"折叠区,可展开回看。
- **时间线条目可展开 AI 分析**:默认一句话摘要折叠;点开看完整层:主题标签 + 业务分析 + 跟进建议 + 需要确认 + 原始截图依据。桌面手机同构。

## 7. 实施顺序(一页一个 PR)

```text
R1  ③待确认 决策卡 + 手机聚焦模式     ← 痛点最重,先做
R2  ①首页今日清单 + 手机底部 Tab 栏
R3  ④客户列表 信息行 + 排序
R4  ⑤客户详情 警示条 + 时间线
R5  ⑥任务 分组(一句话建任务的 AI 解析后置)
R6  ⑦周报 大数字 + 失速信号
```

后置的 AI 项(单独 milestone,需要新后端能力):

- 一句话建任务(文字解析客户名+时间,复用 text extraction 管线)
- 客户绑定自动匹配(AI 提取客户名后 fuzzy match 已有客户,预判"新建/绑定")
- 批量确认(高置信度条目进"全部确认",等 dogfood 置信度数据)
