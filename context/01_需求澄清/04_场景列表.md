## 1. 文档目的

这份文档把“用户真的会怎么用”拆成一个个具体场景。

---

## 2. 第一版核心场景

| 编号 | 场景 | 是否首发 |
|---|---|---|
| S01 | 上传一张聊天截图，系统识别内容并进入待确认队列 | 是 |
| S02 | 手动输入一段文字备注并保存 | 是 |
| S03 | 拍一张名片照片并保存 | 是 |
| S04 | 在待确认队列里确认识别结果 | 是 |
| S05 | 在待确认队列里修改识别错误的信息 | 是 |
| S06 | 跳过某条不想采用的识别结果 | 是 |
| S07 | 打开客户列表浏览全部客户 | 是 |
| S08 | 搜索某个客户并进入详情页 | 是 |
| S09 | 在客户详情页查看时间线 | 是 |
| S10 | 在客户详情页查看待跟进任务 | 是 |
| S11 | 新增一条记录后，系统自动刷新客户的最新跟进状态 | 是 |
| S12 | 系统根据记录生成跟进任务 | 是 |
| S13 | 用户手动补充一个任务 | 是 |
| S14 | 用户把任务标记为完成 | 是 |
| S15 | 查看原始截图 / 原始文字 | 是 |

## 4. 暂不纳入第一版的场景

| 编号 | 场景 | 说明 |
|---|---|---|
| X01 | 通过 Discord 作为正式主入口录入 | 不作为第一版正式路径 |
| X02 | 处理复杂 referral 结算 | 第一版只保留介绍来源标签 |
| X03 | 绘制客户关系图谱 | 不是首发重点 |
| X04 | 多租户隔离 | 未来再说 |
| X05 | 原生 App 分享扩展 | 第一版先不做 |

---

## 5. 当前结论

第一版场景已经足够清晰：重点是单人真实使用闭环，而不是先覆盖所有 CRM 场景。
