# M1 Working Plan

## 1. 目的

这份文档定义 M1 后续可以连续推进的小版本。

它的目标不是写一个大而空的路线图，而是让 Codex 可以在 Chao 不盯着的情况下继续开发低风险事项。

这份文档是自包含的。读这份文档不需要先读其他文档。

## 2. 当前产品边界

当前产品仍然是一个 PWA-first 的个人 CRM side project。

第一用户是 Chao。

第一版核心循环是：

```text
录入原始证据 -> 待确认 -> 确认入库 -> 刷新客户跟进状态 -> 生成或展示任务 -> 下一次跟进
```

当前默认原则：

- 以人为主对象，不以公司为主对象
- review-first，不做自动入库
- local-first，不强制云部署
- single-user first，不先做复杂团队权限
- 不接真实 API key
- 不使用真实客户数据

## 3. 自主推进规则

Codex 可以连续推进以下事项：

- 页面信息架构优化
- 页面 UI 细节优化
- 已有 API 的页面接入
- 已有表结构上的查询和展示
- 测试补充
- 文档和系统日志更新
- 不改变产品边界的小重构

Codex 必须暂停并询问 Chao 的事项：

- 新增或重做 migration
- 删除数据或 drop table
- 使用真实客户数据
- 接入真实 API key
- 改成自动入库
- 改成公司优先
- 改成团队权限优先
- 改成云优先
- 定义沉睡客户、普通客户、重点客户的最终分层规则

## 4. M1.2 客户详情第一屏增强

目标：

把客户详情页从“资料页”改成“销售行动页”。

第一屏必须回答：

- 这个人是谁
- 最近聊了什么
- 下一步该做什么
- 有没有未完成任务
- 为什么现在要跟进

实现范围：

- 使用已有 `parties`、`events`、`attachments`、`tasks` 表
- 展示最新沟通摘要
- 展示原始证据
- 展示 open task
- 展示跟进状态对应的行动提示
- 提供去录入、待确认、任务页的入口

不做内容：

- 不新增字段
- 不新增客户分层规则
- 不自动生成复杂 sales script
- 不接 AI

退出条件：

- 客户详情页第一屏信息层级明显增强
- 页面能展示最新沟通、原始证据和 open task
- `pnpm check` 通过
- `pnpm build` 通过

## 5. M1.3 Review 体验加固

目标：

让待确认页面更适合连续处理多条截图和备注。

实现范围：

- JSON 字段输入错误提示更清楚
- 每条待确认记录更容易区分来源、时间和证据
- confirm 后页面状态更新稳定
- skip 后页面状态更新稳定
- 提供回到客户列表或新增录入的入口

不做内容：

- 不接 OCR
- 不接 Vision API
- 不做自动置信度入库

退出条件：

- 待确认页面可以连续处理多条记录
- 错误信息不需要看控制台也能理解
- `pnpm check` 通过
- `pnpm build` 通过

## 6. M1.4 任务入口补齐

目标：

让任务不仅能展示和完成，也能从页面创建。

实现范围：

- 任务页增加创建任务表单
- 可以选择客户
- 可以填写任务描述
- 可以填写 due date
- 创建后进入 open task 列表

不做内容：

- 不做复杂提醒系统
- 不接日历
- 不做团队分派

退出条件：

- 页面可以创建任务、展示任务、完成任务、恢复任务
- `pnpm check` 通过
- `pnpm build` 通过

## 7. M1.5 本地 Dogfood 指南

目标：

给 Chao 一份可以照着跑的本地测试脚本。

实现范围：

- 写清楚如何启动 Postgres
- 写清楚如何启动 dev server
- 写清楚从新增录入到客户详情的验证路径
- 写清楚如果端口 3000 被占用该怎么办
- 写清楚哪些 demo 数据可以随便改

退出条件：

- Chao 可以不问 Codex，自己跑完整个 M1 demo
- `pnpm check` 通过
- `pnpm build` 通过

## 8. 当前推荐推进顺序

当前推荐顺序：

```text
M1.2 -> M1.3 -> M1.4 -> M1.5
```

原因：

- 客户详情页是 CRM 的决策中心
- Review 页是高频处理入口
- 任务创建是 follow-up 闭环的补强
- Dogfood 指南最后写，才能基于真实页面路径

## 9. 当前风险

当前最大风险不是数据库或框架，而是产品体验变成“能存数据，但不推动行动”。

因此 M1.2 要优先解决：

```text
打开一个客户详情页后，Chao 是否能立刻知道下一步该做什么。
```

如果不能，这个 CRM 就只是通讯录。

M1 的重点不是通讯录，而是跟进系统。
