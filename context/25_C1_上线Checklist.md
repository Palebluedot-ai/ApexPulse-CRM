# C1 上线 Checklist

## 1. 目的

这份文档定义 C1 手机端上线冲刺的上线前检查项。

这份文档是自包含文档。

阅读这份文档不需要依赖其他文档。

C1 上线不是指公开发布给团队。

C1 上线指：

```text
Chao 可以用手机真实记录微信截图，并把客户跟进闭环跑起来。
```

## 2. 必须通过的产品路径

上线前必须跑通：

```text
手机登录 -> 上传微信截图 -> Review -> AI 提取 -> 人工确认 -> 客户详情 -> 任务 -> 周报
```

## 3. 环境检查

必须确认：

- Mac 和手机在同一个 Wi-Fi
- Mac 上 Postgres 正在运行
- Mac 上 Next dev server 正在运行
- 手机可以访问 Network URL
- `.env.local` 已配置视觉 API
- `.env.local` 没有进入 Git
- `data/` 没有进入 Git

## 4. 登录检查

必须确认：

- 登录页可以打开
- 默认开发密码上线前已经改掉，或明确接受只在本地短期 dogfood 使用
- 登录后可以访问 `/capture`
- 未登录访问业务页会跳转登录

推荐上线前改掉：

```text
LOCAL_AUTH_PASSWORD
AUTH_SESSION_SECRET
```

## 5. Capture 检查

必须确认：

- 手机可以选择相册图片
- 上传成功后显示待确认事件 ID
- 上传成功后显示本轮已上传数量
- 上传成功后可以继续上传下一张
- 上传成功后可以进入 Review
- 非图片文件会被拒绝
- 没有选择文件时会提示错误

## 6. Review 检查

必须确认：

- Review 默认只看真实记录
- 可以切换测试记录和全部记录
- 可以看到图片预览
- 可以打开原图
- AI 提取按钮可用
- AI 提取只回填字段，不自动入库
- 页面会显示当前范围待处理数量
- 每张卡片会显示当前是第几条
- 页面会提示确认或跳过后下一条会自动上移
- 可以人工修正客户名、公司、来源、需求、下一步、下次跟进时间
- 可以确认入库
- 可以跳过
- 确认或跳过后，下一条自然上移

## 7. 客户检查

必须确认：

- 新客户可以被创建
- 老客户可以被绑定
- 客户详情显示最新沟通摘要
- 客户详情显示最新证据
- 客户详情显示下一步
- 客户列表能搜索到新客户

## 8. 任务检查

必须确认：

- 填写下一步后会生成 follow-up task
- 下次跟进时间会进入任务 due date
- 任务页可以看到任务
- 任务可以完成
- 任务可以恢复

## 9. 周报检查

必须确认：

- 周报页可以打开
- 本周跟进客户能显示
- 本周待办能显示
- 周报不要求完美文案，但必须能回顾本周动作

## 10. 数据安全检查

必须确认：

- 不提交 `.env.local`
- 不提交 `data/`
- 不提交真实截图
- 不删除真实数据
- 不 drop table
- 不重做 migration
- 不把 AI 结果自动入库

## 11. 工程检查

上线前必须运行：

```bash
pnpm check
pnpm build
```

通过标准：

- lint 通过
- typecheck 通过
- tests 通过
- production build 通过

## 12. 第一轮上线判定

如果满足以下条件，可以认为 C1 第一轮上线可用：

- 手机端 1 张真实截图 E2E 跑通
- 手机端 3 到 5 张连续截图 Review 不迷路
- `pnpm check` 通过
- `pnpm build` 通过
- 没有真实数据误删
- 没有真实密钥进入 Git
- Chao 感觉愿意继续用它记录下一批客户
