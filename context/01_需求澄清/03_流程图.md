## 1. 文档目的

这份文档只描述第一版的关键业务流，不描述代码实现。

---

## 2. 核心录入流程

```mermaid
flowchart TD
    A["用户在 PWA 中上传截图 / 输入文字 / 拍名片"] --> B["系统保存原始内容"]
    B --> C["系统提取可识别字段"]
    C --> D["进入待确认队列"]
    D --> E["用户确认 / 修改 / 跳过"]
    E --> F["生成正式客户记录与时间线事件"]
    F --> G["刷新最近联系时间和最新跟进状态"]
    G --> H["生成或更新待跟进任务"]
```

---

## 3. 客户查看流程

```mermaid
flowchart TD
    A["打开客户列表"] --> B["搜索或选择客户"]
    B --> C["进入客户详情页"]
    C --> D["查看基础信息"]
    C --> E["查看时间线"]
    C --> F["查看待跟进任务"]
    C --> G["查看原始截图 / 原始备注"]
```

---

## 4. 任务形成流程

```mermaid
flowchart TD
    A["用户确认事件信息"] --> B["系统识别下一步 / 跟进建议"]
    B --> C{"是否需要任务"}
    C -- 是 --> D["生成自动任务"]
    C -- 否 --> E["不生成任务"]
    D --> F["刷新下次建议跟进时间"]
    F --> G["用户可补充或手动新建任务"]
```

---

## 5. 未来团队监督流程（预留）

```mermaid
flowchart TD
    A["主管进入团队视图"] --> B["查看团队活动流"]
    B --> C["看每个人今天录了什么"]
    B --> D["看每个人未完成跟进"]
    B --> E["看长期未推进客户"]
```

---

## 6. 当前结论

第一版的流程中心不是“自动化处理”，而是：

- 先顺手录入
- 再人工确认
- 最后沉淀成客户时间线、最新跟进状态和跟进任务
