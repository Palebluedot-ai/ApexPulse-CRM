## 2. 项目定位

项目是 Chao 的 side project。

当前主要场景来自 OTC team 销售跟进，但第一版不按 ApexPulse 内部工具来定性，也不以复杂合规、复杂权限或云部署为前提。

当前唯一真实用户是 Chao。

第一版目标是做一个 PWA-first、local-first、review-first 的个人 CRM。

---

## 3. 已确认的产品原则

### 3.1 PWA-first

第一版正式入口是 PWA，不再沿用 V0 的 Discord-first 入口。

### 3.2 Person-first

第一版以“人”为主对象，不先做公司主对象。

### 3.3 Review-first

自动识别结果不能直接入正式客户档案。  
必须先进入待确认队列，用户可以 confirm / edit / skip。

### 3.4 原始证据必须保留

截图、名片照片、手动文字备注都必须保留原始证据。

### 3.5 跟进状态是核心

系统的核心价值不是“存客户”，而是提醒 Chao 不要让 sales leads 沉睡。

- Vitest
- Drizzle ORM

工程原则：

- 每份文档尽量自包含
- 业务规则先写测试
- 每个小闭环完成后跑验证
- 删除数据、drop table、重做 migration、真实 API key、真实客户数据等动作必须暂停确认
