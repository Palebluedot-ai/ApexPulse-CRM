## 6. M1.13 数据规则

### 6.1 客户绑定规则

Review 确认时有三种情况。

情况一：用户选择已有客户。

系统行为：

- 不创建新客户
- confirmed event 绑定到该客户
- 刷新该客户最新沟通状态
- 如果有下一步，则生成该客户的 followup 任务

情况二：用户没有选择已有客户，但填写了客户名。

系统行为：

- 创建新客户
- confirmed event 绑定到新客户
- 刷新新客户最新沟通状态
- 如果有下一步，则生成新客户的 followup 任务

情况三：用户没有选择已有客户，也没有填写客户名。

系统行为：

- 不创建客户
- confirmed event 不绑定客户
- 不生成客户任务
- 只把 event 标记为 confirmed

原因：

没有客户名时强行创建客户会制造脏数据。

### 6.2 新客户字段规则

创建新客户时字段来源：

- `displayName` 来自 Review 自然字段 `customerName`
- `companyName` 来自 Review 自然字段 `companyName`
- `referralSourceTag` 来自 Review 自然字段 `sourceTag`
- `profileSummary` 来自 Review 自然字段 `needSummary`
- `followupStatus` 默认使用确认时传入的 followupStatus；如果没有传入，则使用 `up_to_date`
- `lastContactAt` 来自 confirmed event 的 `occurredAt`；如果没有 occurredAt，则用 `capturedAt`
- `lastContactSummary` 来自确认摘要
- `lastContactEventId` 来自 confirmed event id
- `nextFollowupAt` 来自 Review 自然字段 `nextFollowupAt`

第一版不写入 tags。

第一版不写入 handlesJson。

### 6.3 自动任务规则

Review 确认时，如果存在最终客户，并且 Review 自然字段 `nextAction` 非空，则创建任务。

任务字段规则：

- `partyId` 使用最终客户 id
- `sourceEventId` 使用 confirmed event id
- `taskType` 使用 `followup`
- `description` 使用 `nextAction`
- `dueAt` 使用 `nextFollowupAt`
- `status` 使用 `open`

如果没有最终客户，不自动创建任务。

如果没有 `nextAction`，不自动创建任务。

### 6.4 时间解析规则

`nextFollowupAt` 是用户输入的自然字段。

第一版只接受 JavaScript `Date` 可以安全解析的字符串。

如果为空：

- party.nextFollowupAt 不更新
- task.dueAt 为空

如果无法解析：

- 不阻塞确认
- 不写入 nextFollowupAt
- 不写入 task.dueAt

原因：

时间格式容错比阻塞用户更重要。

之后如果需要严格日期输入，再把前端控件改成 `datetime-local`。

M2 目标：

支持团队关系和团队可见性。

当前只是排期，不进入 M1。

- M2 不能在 M1.13 中实现

- M2 不能在 M1.15 Weekly report 中顺手实现

- M2 的测试必须覆盖数据可见性，不允许只做 UI
