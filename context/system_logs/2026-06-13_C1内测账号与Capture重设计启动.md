# 2026-06-13 C1 内测账号与 Capture 重设计启动

## 本轮背景

Chao 确认自定义域名已配置成功，正式线上入口可以使用：

```text
https://www.palebluedot.pro
```

随后提出下一阶段需求：下周会有不到 10 个新用户加入内测。当前单账号登录方式不再合适，且 `/capture` 页面需要从原型式表单改成更自然的手机端上传入口。

## 已确认问题

### 1. 内测账号

当前登录仍然是单账号环境变量模式：

```text
LOCAL_AUTH_EMAIL + LOCAL_AUTH_PASSWORD
```

这不适合多人内测。

内测用户不能共用一个账号和密码，否则无法区分上传、Review、客户、任务和反馈来源。

### 2. Capture 页面

当前 `/capture` 页面问题：

- 文字输入和图片输入左右并列，主动作不够聚焦。
- 原生 `Choose File / No file chosen` 暴露在界面中，视觉不一致。
- 未选择图片前出现“保存图片证据”按钮，语义唐突。
- 旧文案仍提到本机 `data/attachments/` 和“先不接 OCR”，已经不符合当前线上 Supabase + Vision 状态。
- Chao 希望页面更像“一个加号即可上传图片”的自然入口。

## 本轮决策

### 1. 先建分支隔离

新分支：

```text
codex/c1-beta-capture-plan
```

目的：

- 不直接在 `main` 上改。
- 先沉淀文档和原型。
- 减少 back and forth 导致主线混乱。

### 2. 先沉淀文档

新增自包含文档：

```text
context/34_C1_内测账号与Capture重设计计划.md
```

该文档包含：

- 内测账号推荐方案。
- Capture 页面重设计目标。
- 临时 `/capture-lab` 原型确认机制。
- to-do list。
- PR 拆分建议。

### 3. 内测账号先用 invite-only，不做自助注册

推荐方案：

- 每个内测用户使用真实公司邮箱。
- 由管理员创建账号和临时密码。
- 暂不做自助注册、忘记密码、邮箱验证码。
- 不进入完整 M2 权限系统。

### 4. Capture 先做临时原型页

推荐先实现：

```text
/capture-lab
```

该页面只用于比较 2 到 3 个上传页方案，不接真实 API，不写数据库。

Chao 选定方案后，再落正式 `/capture` 页面，并删除临时页。

## 当前下一步

建议下一步按顺序执行：

1. 实现 `/capture-lab` 临时原型页。
2. 让 Chao 在手机或浏览器快速确认上传页方向。
3. 根据确认结果重构正式 `/capture`。
4. 再启动 invite-only 多账号登录开发。

## 暂不做

本阶段暂不做：

- 完整团队权限。
- Leader / 下级数据可见性。
- 自助注册。
- 邮箱验证。
- 忘记密码。
- 客户转移。
- 审计日志。

这些属于 M2 或更后面的独立计划。

## 风险提醒

多账号会触碰“单用户 -> 多用户”的产品边界。当前只允许做 C1 内测最小版本，不允许顺手扩展成完整团队权限。

如果后续要做 Leader 查看下级数据，必须单独开 Office Hours 或自包含 M2 权限计划。
