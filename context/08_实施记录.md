### 2.2 技术选择

- 前端框架：Next.js 16
- UI 方向：PWA-first
- 样式：Tailwind CSS 4
- 语言：TypeScript
- 测试：Vitest
- 包管理：pnpm

### 2.3 为什么这样做

Next.js App Router 适合先做 PWA，再逐步增加服务端能力。

Tailwind CSS 适合快速搭建第一版界面，不需要先投入复杂设计系统。

Vitest 适合测试后面的业务规则，例如跟进状态刷新、任务生成、review 状态流转。

pnpm 安装速度快，也能让依赖版本更稳定。

`pnpm build` 已通过。

这代表：

本地 dev server 已验证：

### 2.5 发现并处理的问题

第一次安装依赖时，`eslint@10` 与 Next 生态依赖链存在 peer dependency 风险。

处理方式：

- 固定 Next / React / Tailwind / Vitest 等版本
- 把 ESLint 固定在 Next 生态明确支持的 9.x

第一次生产构建在普通沙盒里失败。

原因：

- Turbopack 构建 CSS 时需要创建子进程并绑定本地端口
- 普通沙盒不允许这个动作

处理方式：

- 使用受控提权重新构建
- 构建通过，说明不是代码问题

Next.js 第一次构建时提示匿名 telemetry。

处理方式：

- 在 `dev`、`build`、`start` 脚本里默认设置 `NEXT_TELEMETRY_DISABLED=1`

## 7. 2026-05-24 Goal Step 2 图片录入保存原始内容

### 7.1 已创建内容

- 图片录入事件和附件构造函数
- 图片录入数据库写入函数
- `POST /api/capture/image`
- 图片录入测试

### 7.2 已落地规则

图片录入会：

- 写入一条 `events` 记录
- 写入一条 `attachments` 记录
- 设置 `source_channel = pwa`
- 设置 `content_type = image`
- 设置 `review_status = pending_review`
- 初始化 `extracted_fields_json = {}`
- 把用户备注保存到 `events.raw_text`

附件记录会保存：

- `storage_key`
- `file_name`
- `mime_type`
- `file_size`
- `width`
- `height`

非图片 `mime_type` 会在写数据库前被拒绝。

空 `storage_key`、空 `file_name`、非正数 `file_size` 会在写数据库前被拒绝。

当前阶段只保存原始证据元数据，不接 OCR，不接 Vision API，不需要外部 API key。

### 7.3 本地 API 验证

本地 dev server 已启动并调用：

```text
POST /api/capture/image
```

```json
  "mimeType": "image/png",

  "width": 1170,
  "height": 2532,
  "note": "API 实测图片备注"
}
```
