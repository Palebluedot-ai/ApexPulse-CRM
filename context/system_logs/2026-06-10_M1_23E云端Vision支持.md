## 原问题

`src/app/api/review/vision-extract/route.ts` 原先只支持：

```text
local-images/
```

并直接使用 `readFile(localAttachmentPath(...))`。

当云端使用：

```text
STORAGE_PROVIDER=supabase
```

图片会保存为：

```text
supabase-images/
```

这会导致 Review 页 AI 提取按钮报：

```text
Local image attachment is required
```

## 本轮改动

新增：

```text
src/server/review/vision-extract-evidence.ts
src/server/review/vision-extract-evidence.test.ts
```

核心规则：

- 缺少图片附件时，返回稳定错误 `Image attachment is required`。
- MIME 不是 `image/*` 时，返回稳定错误 `Image attachment must have an image MIME type`。
- `local-images/` 和 `supabase-images/` 都通过统一的 `readImageEvidence(storageKey)` 读取。
- 不支持的 storage key 保留明确错误 `Unsupported attachment storage key`。

更新：

```text
src/app/api/review/vision-extract/route.ts
```

移除：

```text
storageKey.startsWith("local-images/")
readFile(localAttachmentPath(...))
```

改为：

```text
readExtractableImageEvidence(row.attachment)
```
