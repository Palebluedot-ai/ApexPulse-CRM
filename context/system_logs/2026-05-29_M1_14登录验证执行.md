## 3. 已完成代码

已新增本地登录 session 规则：

- `AUTH_SESSION_COOKIE = otc_crm_session`
- session token 使用 HMAC-SHA256 签名
- session 默认 7 天过期
- session 会校验过期时间
- session 会校验签名，篡改 token 会被拒绝

已新增 Edge middleware 验证：

- middleware 不只检查 cookie 是否存在
- middleware 会在 Edge 环境验证签名和过期时间
- 未登录访问页面会跳转 `/login`
- 未登录访问 API 会返回 `401`

已新增登录页面：

- 路径：`/login`
- 默认显示本地 seed 用户邮箱：`chao.local@example.com`
- 本地默认密码：`local-dev-password`
- 支持 `next` 登录后跳回原页面

已新增登录 API：

- 路径：`POST /api/auth/login`
- 支持 form submit
- 支持 JSON submit
- 登录成功后写入 httpOnly session cookie
- 登录失败返回 `401` 或重定向回 `/login?error=invalid`

已新增退出 API：

- 路径：`POST /api/auth/logout`
- 清除 session cookie
- 重定向回 `/login`

已更新导航：

- 登录后导航显示“退出”

已更新写操作：

- `POST /api/capture/text` 使用 session 用户写入 `createdByUserId`
- `POST /api/capture/image` 使用 session 用户写入 `createdByUserId`
- `POST /api/review/confirm` 使用 session 用户写入 `reviewedByUserId`
- `POST /api/review/skip` 使用 session 用户写入 `reviewedByUserId`
- `POST /api/tasks` 使用 session 用户写入 `createdByUserId`
- `POST /api/tasks/complete` 使用 session 用户写入 `completedByUserId`

## 4. 本地配置

`.env.example` 已补充：

```text
LOCAL_AUTH_EMAIL=chao.local@example.com
LOCAL_AUTH_PASSWORD=local-dev-password
AUTH_SESSION_SECRET=replace-with-a-long-local-random-string
AUTH_STRICT_ENV=false
AUTH_COOKIE_SECURE=false
```

本地 dogfood 默认可以不创建 `.env`。

如果未来部署到云服务器，应设置：

```text
AUTH_STRICT_ENV=true
AUTH_COOKIE_SECURE=true
```
